# SMSV PDF Tools 仕様メモ

## 目的

SMSV の派生として、PDF をサーバーへ送信せず、端末内・ブラウザ内だけで加工できる無料の PWA を作る。

主な用途は次のとおり。

- PDF のページ分割
- ページ回転
- 2-in-1 見開き PDF の左右分割
- 複数 PDF の結合
- ページ単位・範囲単位の一括処理
- PDF の保存ファイル名変更
- PDF 内部メタデータの変更

## 前提条件

- PDF は端末外へ送信しない
- ブラウザ内完結を基本とする
- 商用利用も想定できる無料配布を目指す
- 著作権侵害や真正性の偽装に使えないよう注意文を入れる

## 推奨技術

- フロントエンド: React + TypeScript + Vite
- PDF 表示: PDF.js
- PDF 加工: pdf-lib
- 重い処理の分離: Web Worker
- 配信形態: 静的 PWA
- 配布先候補: Cloudflare Pages などの静的ホスティング

## MVP

最初に入れる機能は次の範囲がよい。

### 読み込み

- PDF のドラッグ & ドロップ
- ファイル選択
- 複数 PDF の読み込み
- ページ数、容量、向き、メタデータの表示

### ページ操作

- 1 ページ抽出
- 範囲抽出
- 奇数ページのみ抽出
- 偶数ページのみ抽出
- ページ削除
- ページ順入れ替え
- ページ順反転
- 90 / 180 / 270 度回転
- 複数 PDF の結合
- 1 ページごとの分割
- 指定ページ数ごとの分割

### 2-in-1 分割

- 左右分割
- 上下分割
- 分割位置の調整
- 左→右 / 右→左 の出力順
- 全ページ一括適用
- 奇数ページのみ / 偶数ページのみ適用

### 名前とメタデータ

- 保存ファイル名のテンプレート変更
- PDF 内タイトルの変更
- Author / Subject / Keywords / Creator / Producer / CreationDate / ModificationDate の変更

## 重要な設計方針

### 1. ファイル名と PDF タイトルを分ける

画面上では次の 2 つを別項目にする。

- 保存ファイル名
- PDF 文書タイトル

### 2. 2-in-1 分割は非破壊クロップを標準にする

初期版は、画質とベクターデータを保ちやすい非破壊クロップを優先する。

再描画方式は後期機能に回す。

### 3. 大量ページは警告する

端末内処理なので、巨大 PDF はブラウザ終了やメモリ不足の可能性がある。

表示目安の例:

- 200 ページ以上: 警告
- 500 ページ以上: 強い警告
- 100 MB 以上: 警告
- 300 MB 以上: 強い警告

### 4. メタデータの真正性には注意書きを入れる

PDF のメタデータは証明書類の真正性を保証しない。
契約書、申請書、証明書などの偽装用途を避ける注意文を表示する。

## ルール処理

大量ページを一定ルールで一括変更できるよう、通常操作と上級ルールを両立させる。

### 画面操作

- 対象ページ
  - 全ページ
  - 奇数ページ
  - 偶数ページ
  - 範囲指定
  - 周期指定
- 操作
  - 回転
  - 分割
  - 削除
  - 並べ替え
  - メタデータ変更

### 内部ルール例

```json
{
  "name": "見開き楽譜を左右分割",
  "steps": [
    {
      "type": "split",
      "target": { "mode": "all" },
      "direction": "vertical",
      "position": 0.5,
      "overlapMm": 2,
      "outputOrder": ["left", "right"]
    },
    {
      "type": "metadata",
      "title": "{originalTitle} - 分割版",
      "filename": "{originalName}_split.pdf"
    }
  ]
}
```

## ファイル名テンプレート

利用例:

- `{originalName}_回転済み.pdf`
- `{originalName}_{startPage}-{endPage}.pdf`
- `楽譜_{index:03}.pdf`

使える変数の候補:

- `{originalName}`
- `{originalTitle}`
- `{index}`
- `{page}`
- `{startPage}`
- `{endPage}`
- `{pages}`
- `{date}`
- `{time}`

## SMSV 連携の考え方

SMSV 向けプリセットを用意する。

- 見開き PDF を単ページ化
- 全ページの向きを統一
- 左右ページサイズを統一
- 不要余白を削除
- 奇数ページなら末尾に空白ページを追加
- タイトルと保存名を整える

初期版は、変換後 PDF を保存して SMSV で読み込む運用が確実。
